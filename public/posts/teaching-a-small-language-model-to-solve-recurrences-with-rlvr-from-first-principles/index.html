Teaching a Small Language Model to Solve Recurrences with RLVR From First Principles

June 12, 2026

We are going to take a language model small enough to run on a single consumer GPU and teach it to solve second-order linear recurrence relations. The tool is RLVR, reinforcement learning from verifiable rewards, and the lab is a math task with a deterministic ground truth.

That ground truth matters. Most reinforcement learning on language models leans on a learned reward model, a second network that estimates how good an answer is. Those estimates can be gamed. A recurrence relation does not negotiate: either a proposed closed form reproduces the sequence when we plug in n=0,1,2,…n = 0, 1, 2, \dotsn=0,1,2,…, or it does not. That makes the task a clean instrument for watching how each RLVR algorithm changes the policy-gradient update.

The point is not to build a leaderboard of increasingly ornate losses. GRPO, GSPO, DAPO, CISPO, and Dr. GRPO each repair a different failure mode in the same basic machinery. This post asks which failures actually show up in a tiny recurrence lab on Llama-3.2-1B-Instruct, and which fixes are aimed at problems this run does not yet have.

From Pretraining To Verifiable Fine-Tuning#

A language model is a tokenizer plus a stack of layers that assigns probabilities to the next token. Pretraining fills that stack with broad fluency: math-like strings, algebraic patterns, and the habit of continuing text. It does not, by itself, teach a reliable multi-step scratchpad or a verifier-friendly answer format. Instruct tuning adds conversational obedience and short-answer habits, but still leaves the architectural limits we spell out in the appendix.

Our pipeline has three data stages on top of the base instruct checkpoint. Set A holds gold-trace supervised examples that teach the four-step recurrence protocol and <ans> tags. Set B is the RL training pool: the same prompt distribution, but learning comes from sampled completions and verifier scores. Set C is held out for evaluation only, disjoint in root pairs and problem instances so we measure generalization rather than recall. Every number in the results section comes from set C.

Most production RL on language models still uses a learned reward model: a second network trained to score answers. That works at scale, but the reward model can be gamed and its mistakes become the optimization target. RLVR replaces the learned judge with a deterministic check. On tasks like code, formal math, and structured recurrences, the verifier is cheap, repeatable, and not subject to preference drift. That is why RLVR is a promising path in regimes where the ground truth is machine-checkable, even though it does not replace pretraining or supervised format teaching.

The full architectural motivation and step-by-step recurrence pipeline are in the appendix.

The Recurrence Lab#

The task is deliberately narrow. Each prompt gives a second-order linear recurrence,

an=A an−1+B an−2, a_n = A\,a_{n-1} + B\,a_{n-2}, an​=Aan−1​+Ban−2​,

along with a0a_0a0​ and a1a_1a1​. The model must produce a closed form for ana_nan​ inside inline <ans> tags, using nnn as the variable. A verifier extracts the tagged expression, evaluates it for the first few indices, and compares those values against the true sequence.

This is a useful RLVR lab for two reasons. First, the reward is deterministic and cheap: no human labels, no learned judge, no partial-credit model pretending to understand algebra. Second, the problem family is enumerable. We can generate the whole universe, split train and evaluation cleanly, and know when a result is measuring generalization rather than recall.

The caveat is just as important. This is not a benchmark for mathematical reasoning in the wild. The traces are short, the roots are clean integers, and the held-out set is small. We use the task as a mechanism lab: small enough that the reward variance, ratio variance, clipping, and dead-group behavior can be plotted one batch at a time.

Building The Lab#

We generate the dataset by running the math backward. Instead of sampling random recurrence coefficients and hoping their characteristic roots are clean, we choose two distinct integer roots from {−4,−3,−2,2,3,4,5}\{-4, -3, -2, 2, 3, 4, 5\}{−4,−3,−2,2,3,4,5} and two nonzero integer constants C1,C2C_1, C_2C1​,C2​. The closed form is fixed first:

an=C1r1n+C2r2n. a_n = C_1 r_1^n + C_2 r_2^n. an​=C1​r1n​+C2​r2n​.

Vieta’s formulas then give the recurrence coefficients,

A=r1+r2,B=−r1r2,A = r_1 + r_2, \qquad B = -r_1 r_2,A=r1​+r2​,B=−r1​r2​,

and the first two sequence values give the initial conditions,

a0=C1+C2,a1=C1r1+C2r2.a_0 = C_1 + C_2, \qquad a_1 = C_1 r_1 + C_2 r_2.a0​=C1​+C2​,a1​=C1​r1​+C2​r2​.

With seven roots there are 212121 distinct root pairs. With six nonzero choices for each constant, the universe before guards is about 21×36≈70021 \times 36 \approx 70021×36≈700 problems. We resample zero constants and degenerate-looking zero initial values, then split roots and whole problem instances so train and evaluation are disjoint.

That smallness is a feature, but it imposes discipline. If train and eval share the same problems, the verifier will reward memorized formulas. If the traces are too long to inspect, the loss mechanics disappear into aggregate curves. Here the derivations are roughly 380380380 tokens, which keeps the experiment in the range where a single batch can still be read as evidence about a mechanism.

From Policy Gradients To Tokens#

RLVR touches one surface of the language model: the log-probability of the tokens the model actually sampled. Treat each sampled token as an action, the completion as a trajectory, and the verifier score as a return that arrives at the end. The policy-gradient theorem tells us to push on ∇θlog⁡πθ(ot∣o<t)\nabla_\theta \log \pi_\theta(o_t \mid o_{\lt t})∇θ​logπθ​(ot​∣o<t​), weighted by how good the trajectory was.

Two refinements make that usable. The first is a baseline. Instead of weighting tokens by raw reward, we weight them by an advantage, the reward relative to some reference. The second is a trust region. We sample completions from an old policy and update the current policy on those samples, so we correct the off-policy gap with an importance ratio,

ri,t=exp⁡ ⁣(log⁡πθ(oi,t∣oi,<t)−log⁡πθold(oi,t∣oi,<t)).r_{i,t} = \exp\!\left(\log \pi_\theta(o_{i,t} \mid o_{i,\lt t}) - \log \pi_{\theta_{\mathrm{old}}}(o_{i,t} \mid o_{i,\lt t})\right).ri,t​=exp(logπθ​(oi,t​∣oi,<t​)−logπθold​​(oi,t​∣oi,<t​)).

Left unchecked, one token whose probability moved too far can dominate the update. PPO-style clipping keeps the ratio inside a band around 111. The GRPO family keeps rearranging the same three objects: the advantage, the ratio, and the clip.

Why GRPO Had To Exist#

RLVR did not appear from nowhere. One earlier route was bootstrapping from successes, in the STaR and ReST lineage: let the model attempt problems, keep the correct traces, and fine-tune on them. That assumes the base model can already solve some prompts. A 1B model on this recurrence task mostly cannot, so the cold-start problem arrives before learning begins.

Full PPO solves exploration more directly by learning a value function for partial trajectories. The cost is a critic, often another model-sized network. On a single consumer GPU, that extra memory can be the difference between a run that fits and a run that never starts.

GRPO keeps PPO’s clipped policy-gradient update and removes the critic. For each prompt, sample a group of completions, score each one with the verifier, and let the group itself provide the baseline. Relative to its siblings, was this answer above or below average?

GRPO: Advantages From The Group#

For a prompt with GGG completions and scalar rewards R1,…,RGR_1,\dots,R_GR1​,…,RG​, canonical GRPO uses the group-normalized advantage

A^i=Ri−RˉσR+ϵ,Rˉ=1G∑j=1GRj.\hat{A}_i = \frac{R_i - \bar{R}}{\sigma_R + \epsilon}, \qquad \bar{R} = \frac{1}{G}\sum_{j=1}^G R_j.A^i​=σR​+ϵRi​−Rˉ​,Rˉ=G1​j=1∑G​Rj​.

A completion that solved the recurrence when most siblings failed receives a positive advantage. A completion that failed when siblings solved it receives a negative one. The group mean is the baseline that a critic would otherwise have to learn.

The standard-deviation denominator is a modeling choice. It rescales every group to comparable spread, but it also makes the effective gradient weight depend on how dispersed the rewards happened to be. Dr. GRPO removes that denominator and keeps a std-free group baseline,

A~i=Ri−Rˉ.\tilde{A}_i = R_i - \bar{R}.A~i​=Ri​−Rˉ.

That variant targets difficulty and length-normalization artifacts. In this lab it is a useful diagnostic arm, not a rescue.

With advantages in hand, GRPO applies the clipped surrogate token by token:

LGRPO=−1B∑i=1B1Ti∑t=1Timi,t min⁡ ⁣(ri,tA^i, clip(ri,t,1−εlo,1+εhi) A^i).\mathcal{L}_{\mathrm{GRPO}} = -\frac{1}{B}\sum_{i=1}^{B}\frac{1}{T_i}\sum_{t=1}^{T_i} m_{i,t}\,\min\!\left(r_{i,t}\hat{A}_i,\,\mathrm{clip}(r_{i,t}, 1-\varepsilon_{\mathrm{lo}}, 1+\varepsilon_{\mathrm{hi}})\,\hat{A}_i\right).LGRPO​=−B1​i=1∑B​Ti​1​t=1∑Ti​​mi,t​min(ri,t​A^i​,clip(ri,t​,1−εlo​,1+εhi​)A^i​).

For plain GRPO we use εlo=εhi=0.2\varepsilon_{\mathrm{lo}} = \varepsilon_{\mathrm{hi}} = 0.2εlo​=εhi​=0.2. The outer sum averages over completions in the batch; the inner sum averages over tokens in completion iii, with mi,tm_{i,t}mi,t​ masking padding. The important shape is the mismatch: A^i\hat{A}_iA^i​ is one number for the whole completion, but ri,tr_{i,t}ri,t​ is a token-level ratio. A 300-token derivation gets the same verifier signal broadcast over every token, while the importance ratio varies token by token.

We can see that geometry in the 1B training logs. The figure below tracks the fraction of prompt groups with zero reward variance across training steps for each variant on Llama-3.2-1B. Two observations match the GRPO story. Group contrast is the live signal: reward std stays nonzero only when siblings disagree. Clipping is quiet on these short traces, so the scatter plot of per-token ratios is not the bottleneck here.

Verdict here: GRPO’s critic-free group contrast is load-bearing, while its token-level clipping pathology barely bites on short traces.

GSPO: Move The Ratio To The Sequence#

GSPO takes the mismatch literally. If the verifier reward is sequence-level, make the importance ratio sequence-level too. It sums token log-ratios across the completion, divides by length, and exponentiates once:

ρi=exp⁡ ⁣(1Ti∑t=1Timi,t[log⁡πθ(oi,t∣oi,<t)−log⁡πθold(oi,t∣oi,<t)]).\rho_i = \exp\!\left(\frac{1}{T_i} \sum_{t=1}^{T_i} m_{i,t} \left[\log \pi_\theta(o_{i,t}\mid o_{i,\lt t}) - \log \pi_{\theta_{\mathrm{old}}}(o_{i,t}\mid o_{i,\lt t})\right]\right).ρi​=exp(Ti​1​t=1∑Ti​​mi,t​[logπθ​(oi,t​∣oi,<t​)−logπθold​​(oi,t​∣oi,<t​)]).

The clipped surrogate then uses ρi\rho_iρi​ everywhere ri,tr_{i,t}ri,t​ appeared:

LGSPO=−1B∑i=1B1Ti∑t=1Timi,t min⁡ ⁣(ρiA^i, clip(ρi,1−εlo,1+εhi) A^i).\mathcal{L}_{\mathrm{GSPO}} = -\frac{1}{B}\sum_{i=1}^{B}\frac{1}{T_i}\sum_{t=1}^{T_i} m_{i,t}\,\min\!\left(\rho_i\hat{A}_i,\,\mathrm{clip}(\rho_i, 1-\varepsilon_{\mathrm{lo}}, 1+\varepsilon_{\mathrm{hi}})\,\hat{A}_i\right).LGSPO​=−B1​i=1∑B​Ti​1​t=1∑Ti​​mi,t​min(ρi​A^i​,clip(ρi​,1−εlo​,1+εhi​)A^i​).

In our implementation the GSPO clip window is tiny, εlo=3×10−4\varepsilon_{\mathrm{lo}} = 3 \times 10^{-4}εlo​=3×10−4 and εhi=4×10−4\varepsilon_{\mathrm{hi}} = 4 \times 10^{-4}εhi​=4×10−4, because sequence-averaged ratios concentrate near 111.

On the 1B runs, completion-level clipping stays near zero throughout training. The figure below plots TRL’s completions/clipped_ratio metric for each variant. At roughly 380380380 tokens per trace, the token-level ratios rarely leave the band, so GSPO’s variance reduction has little room to change the outcome.

That variance reduction is still the point of GSPO on very long chains of thought or stale off-policy samples, where token-level ratios can dominate the update. In this recurrence lab, though, traces are short and the clip fraction stays near zero.

Verdict here: GSPO fixes a real sequence-ratio failure mode, but this run mostly shows the mechanism rather than a payoff.

DAPO: Widen The Upper Clip, Drop Dead Groups#

DAPO is best read as two practical interventions on top of GRPO. The first is clip-higher: keep the lower clip at 0.20.20.2, but widen the upper clip so positive-advantage tokens have more room to increase in probability:

εlo=0.2,εhi=0.28.\varepsilon_{\mathrm{lo}} = 0.2, \qquad \varepsilon_{\mathrm{hi}} = 0.28.εlo​=0.2,εhi​=0.28.

This is separate from dynamic sampling. Clip-higher changes the surrogate geometry. In our diagnostics it does exactly what the geometry predicts: widening the upper bound roughly halves the share of positive-advantage tokens clipped on the high side, from about 0.25%0.25\%0.25% to about 0.14%0.14\%0.14%. That mechanism fires, but it does not imply the final reward must improve when clipping is already rare.

The second intervention is dynamic sampling. Under group-relative advantages, a prompt group with all completions correct or all completions wrong carries no contrast. Its reward variance is zero, so every centered advantage is zero:

1 ⁣[std(Rg,1,…,Rg,G)≥ϵ].\mathbf{1}\!\left[\mathrm{std}(R_{g,1},\ldots,R_{g,G}) \ge \epsilon\right].1[std(Rg,1​,…,Rg,G​)≥ϵ].

Full DAPO resamples until the batch is informative. Our implementation uses the cheaper tutorial version: zero the advantages for zero-variance groups rather than drawing replacements.

One captured group makes the issue concrete:

This is an informative group. The two format-only completions are pushed down and the six correct completions get a smaller lift. If all eight completions had the same reward, the update from this prompt would vanish. Across the logged runs, the sampled prompt group tied on roughly 70%70\%70% of steps, so dynamic sampling addresses a failure mode that clearly appears.

Verdict here: clip-higher has measurable local effect but little room to matter; dynamic sampling is load-bearing because dead groups are common.

CISPO: Keep A Bounded Gradient Alive#

CISPO targets a side effect of PPO-style clipping. In the GRPO surrogate, once a token’s ratio crosses the clipping boundary in the active direction, the surrogate can stop passing gradient through that token. If clipping is frequent, rare recovery tokens could be silenced just when they need reinforcement.

CISPO changes the form of the update. It clamps the importance weight, detaches that weight, and multiplies it into the log-probability:

wi,t=sg ⁣(min⁡ ⁣(ri,t,cmax⁡)),LCISPO=−∑i,tmi,t wi,tA^ilog⁡πθ(oi,t∣oi,<t)∑i,tmi,t.w_{i,t} = \mathrm{sg}\!\left(\min\!\left(r_{i,t}, c_{\max}\right)\right), \qquad \mathcal{L}_{\mathrm{CISPO}} = -\frac{\sum_{i,t} m_{i,t}\, w_{i,t}\hat{A}_i \log \pi_\theta(o_{i,t}\mid o_{i,\lt t})}{\sum_{i,t} m_{i,t}}.wi,t​=sg(min(ri,t​,cmax​)),LCISPO​=−∑i,t​mi,t​∑i,t​mi,t​wi,t​A^i​logπθ​(oi,t​∣oi,<t​)​.

The weight caps the size of the update, but the log-probability term still has a gradient path. The usual story is that this may preserve learning signal for rare self-correction tokens. In our run that remains a hypothesis suggested by the loss geometry rather than a directly measured token-level behavior, because clipping touches only a fraction of a percent of tokens.

Verdict here: CISPO fixes clipped-token silence when clipping is active; in this short-trace lab the relevant pathology is mostly dormant.

Reading One Prompt#

Aggregate numbers hide the texture. On the prompt an=16 an−2a_n = 16\,a_{n-2}an​=16an−2​ with a0=−1a_0 = -1a0​=−1 and a1=−20a_1 = -20a1​=−20, the base model never reaches the protocol. It invents the wrong characteristic equation, carries that error forward, and emits no parseable <ans> block. The verifier scores it 0.00.00.0.

The SFT model learns the format. It writes the characteristic-equation step, factors, builds a system, and emits a tagged answer. But it factors r2−16=0r^2 - 16 = 0r2−16=0 as if the roots were 444 and 555 instead of 444 and −4-4−4. The scaffold is right, the algebra is wrong, and the verifier gives format credit only: 0.10.10.1.

The GRPO model gets this sample right. It factors r2−16r^2 - 16r2−16 into (r+4)(r−4)(r+4)(r-4)(r+4)(r−4), solves the constants, and emits the closed form an=2(−4)n−3⋅4na_n = 2(-4)^n - 3\cdot 4^nan​=2(−4)n−3⋅4n, which reproduces −1,−20,−16,−320,−256-1, -20, -16, -320, -256−1,−20,−16,−320,−256. One solved prompt is not a trend. It is a concrete look at what the reward is pushing toward: the same four-step scaffold as SFT, with the algebra inside the scaffold landing more often.

Which Failure Mode Should You Pay To Fix?#

The practical lesson is not to always use GRPO. Name the pathology before paying for the fix. In this 1B lab the active pathology is sparse within-group signal on short traces.

Reach for GRPO when you want the simplest critic-free update and your sampled solutions are short enough that token-ratio variance and clipping are quiet. That is this recurrence setting: roughly 380380380-token traces, near-zero clip fraction, and enough mixed groups for group-relative advantages to carry the learning signal.

Add dynamic sampling when groups are dying. If most prompt groups are all wrong early in training, or all right after the policy locks in, the group-centered advantage is zero and the batch wastes gradient budget. Filtering or resampling for contrast keeps sparse verifier bits turning into updates.

Reach for GSPO when the sequence-level mismatch is actually hurting you. Long chain-of-thought runs, stale samples, and off-policy pipelines can make token-level ratios volatile. The recurrence run teaches the GSPO mechanism, but the corrected Set C result is mechanism plus extrapolation, not a win: pass@1 0.2920.2920.292 and pass@4 0.4580.4580.458.

Reach for clip-higher or CISPO when clipping is active enough to bias learning. Clip-higher helps when positive-advantage tokens are being capped too conservatively. CISPO helps when clipped tokens still need bounded gradient signal. In this run, clipping barely fires, so those fixes have little surface area. Clip-higher still reaches pass@1 0.4170.4170.417 and pass@4 0.6670.6670.667, while CISPO’s advantage remains mostly a lesson from the objective rather than from the recurrence numbers.

Dr. GRPO fits the same diagnostic pattern. It targets length bias and standard-deviation normalization artifacts. Those are plausible failures to look for as traces lengthen, but here dropping std normalization does not beat plain GRPO: pass@1 0.3230.3230.323 and pass@4 0.4580.4580.458 at step 60.

The frontier is not a ladder of losses. It is a map from observed failure modes to interventions: critic variance, dead groups, sequence-ratio variance, over-conservative clipping, clipped-token silence, and length or normalization bias. Measure which one is biting, then choose the smallest method that fixes it.

Two Ceilings#

There is one more question underneath the loss comparison. When RLVR improves a small model, is it expanding the set of solutions the model can reach, or merely putting more probability on solutions that SFT already made available?

Best-of-kkk gives a useful probe. At a matched budget of pass@8 on 24 prompts, SFT reaches 0.450.450.45, while the RL checkpoint reaches 0.720.720.72. That does not prove a broad theorem about RLVR, but in this controlled setting it shows that RL is not only making the greedy sample cleaner. It is making correct derivations more reachable under sampling.

The second ceiling is harder. We also tested repeated-root recurrences, where the right template is no longer C1r1n+C2r2nC_1 r_1^n + C_2 r_2^nC1​r1n​+C2​r2n​ but a doubled-root form such as (C1+C2n)rn(C_1 + C_2 n)r^n(C1​+C2​n)rn. The distinct-root training template cannot express that procedure, and the transfer result is blunt: 0/960/960/96. This is not a best-of-kkk failure, not a GSPO-versus-GRPO failure, and not a problem a wider clip will solve. The model is being asked for a procedure the data distribution did not teach.

Put the two ceilings together and the lesson gets sharper. RL can widen the reachable part of the solution distribution inside a learned procedure. It does not synthesize a new algebraic template when the training world never made that template available.

Takeaway: Diagnose, Then Optimize#

The base model failed our recurrence task because it lacked room and second chances: a saturated residual stream, a fixed per-token compute budget, and no backspace. SFT taught it to write the four-step scratchpad into context. RLVR then used a deterministic verifier to reward traces where the algebra inside that scratchpad worked.

The corrected outcome is useful because it refuses the tidy ladder story. GRPO wins the small 1B five-arm comparison because, here, critic-free group contrast and informative sampled groups matter more than sequence-level clipping pathologies. GSPO, clip-higher, CISPO, and Dr. GRPO remain meaningful methods, but their value depends on the failure mode appearing in the run being trained.

The final mental model is simple. Start with the verifier and sampled groups. Check whether groups contain contrast. Check whether ratios are exploding, whether clipping is active, whether rare recovery tokens are being silenced, and whether length or normalization biases are visible. Pay for the method whose pathology is present. On this tiny recurrence lab, that diagnostic discipline is why the simplest method wins.

Appendix: The Recurrence Task In Detail#

The Invisible Wall: Why Small Models Break On Multi-Step Algebra#

Picture handing the following prompt to a freshly downloaded 1B-parameter instruct model:

Find the closed-form expression for the sequence ana_nan​ that satisfies the recurrence relation an=−5 an−1−6 an−2a_n = -5\,a_{n-1} - 6\,a_{n-2}an​=−5an−1​−6an−2​ for n≥2n \ge 2n≥2, given the initial conditions a0=−4a_0 = -4a0​=−4 and a1=10a_1 = 10a1​=10.

There is exactly one correct closed form here, and a prepared student can find it with pencil and paper. The model, left alone, usually cannot. Three different architectural pressures show up at once.

Crisis A: Compute-As-You-Speak#

Transformers are autoregressive. They emit one token at a time, and the compute budget for producing each token is a single fixed forward pass through a fixed stack of layers. There is no dial the model can turn to think harder about a hard token.

If we demand the answer immediately, with no reasoning in between, the model has to set up the characteristic equation, factor it, build the 2×22 \times 22×2 linear system from the initial conditions, and solve for two constants inside the hidden state of a single next-token prediction. A 1B model does not have much room to fold that whole pipeline into one pass. Starved of external scratch space, it tends to pattern-match a formula that resembles answers it saw during pretraining.

Crisis B: Residual Stream Saturation#

When a person solves this on paper, the paper is working memory. You write down a root, then a system, then a constant, and those marks stay stable while the next step happens.

A transformer’s internal scratchpad is its residual stream, the vector that information rides through the layers. In a small model that vector is cramped and has many jobs at once. It has to track syntax, formatting constraints, the initial values, the roots, and the constants. Pack too many live quantities into one constrained vector and they interfere. A root computed a few tokens earlier can degrade because the same representational space is now serving another part of the derivation.

Crisis C: The Tyranny Of The Next Token#

There is no backspace. A standard language model cannot stop, notice that a line does not add up, and erase it. Each new line conditions on the text so far, including the model’s own mistakes.

If the model flips a sign while setting up the constant-solving system, the next line is built on top of the wrong equation, and the line after that inherits the same error. A correct trajectory might run

r1=3, r2=−2  ⟹  C1+C2=2,r_1 = 3,\ r_2 = -2 \;\Longrightarrow\; C_1 + C_2 = 2,r1​=3, r2​=−2⟹C1​+C2​=2,

while a single sign error can send it down

r1=3, r2=−2  ⟹  C1−C2=2.r_1 = 3,\ r_2 = -2 \;\Longrightarrow\; C_1 - C_2 = 2.r1​=3, r2​=−2⟹C1​−C2​=2.

Once it leaves the valid path, a base model often continues confidently. The failure is not a lack of math vocabulary. It is a lack of room, stable scratch space, and second chances.

The Recurrence Problem Step By Step#

A second-order linear recurrence is the discrete cousin of a second-order differential equation. Every term is a fixed linear combination of the two before it:

an=A an−1+B an−2. a_n = A\,a_{n-1} + B\,a_{n-2}. an​=Aan−1​+Ban−2​.

We want a closed form, an expression where we plug in nnn and read off ana_nan​ directly. The human pipeline has four stages.

Step 1: characteristic equation. Guess that solutions look like an=rna_n = r^nan​=rn and substitute into the recurrence. Dividing through by rn−2r^{n-2}rn−2 collapses the relation into a quadratic:

r2−A r−B=0. r^2 - A\,r - B = 0. r2−Ar−B=0.

Step 2: factor and find the roots. Solve the quadratic for roots r1r_1r1​ and r2r_2r2​. In this dataset the roots are clean integers, so the step is a factorization rather than a messy quadratic-formula exercise. A sign error here propagates through every later computation.

Step 3: write the general solution and initial-condition system. With distinct roots, the general solution is

an=C1r1n+C2r2n. a_n = C_1 r_1^n + C_2 r_2^n. an​=C1​r1n​+C2​r2n​.

The constants C1C_1C1​ and C2C_2C2​ are pinned down by a0a_0a0​ and a1a_1a1​:
